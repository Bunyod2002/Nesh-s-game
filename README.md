# Nesh-s-game
Simulation for the prisoner's dilemma

# Итеративная дилемма заключённого

Этот проект представляет собой симуляцию **итеративной дилеммы заключённого** — классической задачи теории игр, где два игрока многократно выбирают между сотрудничеством и предательством, стремясь максимизировать свои очки. Программа позволяет тестировать различные стратегии (классы) в турнире на 100 раундов.

## 📜 Правила игры

В каждом раунде оба игрока одновременно делают ход:
- **0** — сотрудничество (мирный ход)
- **1** — предательство (агрессивный ход)

Очки начисляются по следующей матрице:

| Ход игрока A | Ход игрока B | Очки A | Очки B |
|--------------|--------------|--------|--------|
| 0 (мир)      | 0 (мир)      | +3     | +3     |
| 0 (мир)      | 1 (агрессия) | 0      | +5     |
| 1 (агрессия) | 0 (мир)      | +5     | 0      |
| 1 (агрессия) | 1 (агрессия) | +1     | +1     |

Цель — набрать как можно больше очков за всю серию раундов.

## 🗂 Структура проекта

- `prisoners.py` — содержит абстрактный базовый класс `Builder` и три готовые стратегии:
  - `Peace` — всегда сотрудничает (возвращает 0)
  - `Agression` — всегда предаёт (возвращает 1)
  - `Eye_for_eye` — стратегия «зуб за зуб» (начинает с мира, затем копирует последний ход противника)
- `main.py` — запускает турнир между двумя игроками (в текущей версии — `Agression` и `Peace`). Вы можете изменять пару стратегий по своему усмотрению.

## 🚀 Как запустить

1. Убедитесь, что у вас установлен Python 3.6 или выше.
2. Скачайте оба файла (`main.py` и `prisoners.py`) в одну папку.
3. Отредактируйте `main.py`, чтобы выбрать нужные стратегии. Например:
   ```python
   player1 = pr.Eye_for_eye()
   player2 = pr.Peace()

Чтобы добавить новую стратегию, создайте класс, наследующий от Builder (из prisoners.py), и реализуйте метод move(self, moves). Этот метод должен возвращать целое число: 0 (сотрудничество) или 1 (предательство).

В конструкторе (__init__) вы можете инициализировать любые переменные, которые будут хранить состояние стратегии (например, флаг обиды, счётчик повторений, последний ход и т.п.).
